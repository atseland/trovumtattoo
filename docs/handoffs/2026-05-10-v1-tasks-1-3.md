   - `bookings` har `archivedAt`, `archivedBy` og `archiveReason`.
   - Admin kan arkivere kun etter `completed` eller `cancelled`.
   - Arkiverte bookinger filtreres bort fra aktive prosjekt-, kommende-, dashboard- og notification-flater.
   - Prosjektdetalj har aktiv/arkiv-filter, restore og permanent delete fra arkiv.

2. Customer-scoped new mail:
   - Ny e-post starter fra kundedetalj, ikke global mailflate.
   - Convex-actionen tar `clientId`, henter eksisterende kunde, bruker kundens e-post som eneste mottaker og lagrer outbound mail som `mailThreads` + `mailMessages` etter SMTP-suksess.
   - SMTP-feil kastes videre og gir ikke falsk suksess eller lagret tråd.
